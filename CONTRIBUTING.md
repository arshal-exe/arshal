# Contributing

Thank you for your interest in contributing! This repository is a personal portfolio and contributions are welcome via issues and pull requests.

Guidelines:

- Issues: Open an issue for bugs or feature suggestions using the provided templates.
- PRs: Fork the repository, create a descriptive branch, and submit a pull request. Keep changes small and focused.
- Tests: There are no automated tests in this repository; please include screenshots and reproduction steps for visual changes.
- Code style: Keep changes consistent with existing HTML/CSS/JS style. Use semantic HTML and accessible markup when possible.

Process:

1. Open an issue describing the change or feature.
2. If approved, create a branch `feat/your-feature` or `fix/your-fix`.
3. Submit a Pull Request referencing the issue.

See `CODE_OF_CONDUCT.md` for expected community behavior.
